# Full-Stack Developer | 성인석

**"처음부터 끝까지 서비스를 만들어내는 개발자"**

모던 웹 기술 스택을 활용한 풀스택 개발과 클라우드 인프라 구축 경험을 바탕으로,  
안정적이고 확장 가능한 서비스를 구현하는 개발자입니다.  
팀 프로젝트 리더 경험을 통해 기술적 역량과 커뮤니케이션 능력을 균형 있게 발전시켜 나가고 있습니다.

#### 주요 기능

- 모듈형 아키텍처 설계 (NestJS의 모듈 시스템 활용)
- JWT 인증 시스템 및 보안 구현
- Docker 컨테이너 기반 개발/배포 환경 구축

#### 배운 점
- **모던 풀스택 아키텍처**: Next.js 15와 NestJS를 활용한 TypeScript 기반 개발
- **모듈형 설계**: 기능별 독립적인 모듈로 확장성과 유지보수성 확보
- **컨테이너 기반 개발**: Docker 핫리로드와 헬스체크 시스템 구현
- **보안 시스템**: JWT 토큰 인증과 bcrypt 해싱을 통한 안전한 사용자 관리
- **TypeScript 활용**: 런타임 에러 방지 및 타입 안전성 확보



#### 기억에 남는 이슈 해결 or 작업 경험
- **Docker 핫리로드 설정 이슈**  
  → 개발 환경에서 코드 변경 시 자동 재시작이 되지 않는 문제 발생  
  → `CHOKIDAR_USEPOLLING=true`와 `WATCHPACK_POLLING=true` 설정으로 파일 변경 감지 방식을 폴링으로 변경하여 해결

- **다중 관리자 환경에서의 동시성 제어 및 트랜잭션 처리**  
  → 팀원 관리 페이지에서 드래그앤드롭으로 팀원 소개 순서를 변경할 때, 여러 관리자가 동시에 작업하면서 순서 데이터 불일치 문제 발생  
  → TypeORM의 낙관적 락(Optimistic Locking)을 통해 버전 기반 충돌 감지 구현, 동시 수정 감지 시 최신 데이터로 자동 갱신 및 사용자 알림 제공

- **액세스/리프레시 토큰 기반 자동 인증 갱신 시스템**  
  → 액세스 토큰 만료 시마다 사용자가 재로그인해야 하는 불편함과 보안 취약점 발생  
  → 쿠키 기반 리프레시 토큰 저장 및 액세스 토큰 자동 갱신 로직 구현으로 사용자 경험과 보안성을 동시에 확보

### 2. LostParty (팀 프로젝트 | 팀 리더)  
> 실시간 게임 파티 정보 제공 웹서비스

- **팀 구성**: 3명 (백엔드/프론트엔드/OCR)

`Node.js`, `Express.js`, `MySQL`, `Docker`, `AWS EC2`, `Swagger`, `Git`, `Notion`

#### 주요 기능
- OCR로 게임 화면에서 닉네임 인식 → 블랙리스트 확인
- LostArk API 연동 → 캐릭터 정보 조회
- Swagger를 활용한 API 문서 자동화 및 프론트 협업
- Docker 환경 구성 및 EC2 실서버 배포

#### 내 역할
- 팀 리더 및 백엔드 개발 담당
- 프로젝트 구조 설계 및 전반적인 배포 환경 구성
- 서버 API 구축, AWS EC2 환경 세팅, 도메인 연결, SSL 인증, 로스트아크 API 연동 수행

#### 배운 점
- Docker로 개발환경을 통합하고 실배포까지 경험  
- Swagger를 활용한 API 문서화 및 프론트와의 협업  
- 팀원들의 상황 체크 및 개발 속도 조율로 **리더십 경험 강화**  
- **Git으로 코드 형상 관리 및 기능 분담**, PR 통한 리뷰 및 공유 경험



#### 기억에 남는 이슈 해결 or 작업 경험
- **HTTPS 연결 이슈 (SSL 인증)**  
  → 인증서는 발급되었지만 Nginx 설정 오류로 인해 브라우저에서 “안전하지 않음” 경고가 발생  
  → Certbot을 활용해 SSL 인증서를 재설정하고, `nginx.conf`에서 `ssl_certificate` 경로 및 HTTPS 리다이렉션 설정을 올바르게 구성하여 해결  

- **로컬 개발 환경과 실제 서비스 환경의 차이에서 오는 개념적 어려움**  
  → 그동안 `localhost`에서만 개발하다 보니, “다른 사람은 어떻게 접속하지?”라는 의문과 함께 실제 서비스 배포의 전체 흐름을 이해하는 데 어려움이 있었음  
  → EC2 서버 구축, 포트 개방, 도메인 연결, Nginx 설정, HTTPS 발급 등 웹 서비스의 “운영” 관점에서 필요한 절차를 직접 경험하며 개념을 체득함  

- **Nginx reverse proxy 설정 이슈**  
  → 프론트엔드에서 API 요청이 제대로 전달되지 않거나, 서버 간 충돌이 발생  
  → `/api` 경로 설정 누락, 프록시 포트 중복 문제 등을 반복적으로 점검하며 각 요청 흐름을 정리하고, 명확한 포트 분리와 proxy 설정을 통해 안정화

## 기억에 남는 프로젝트
‘로스트파티’는 게임 화면을 실시간으로 인식하여 파티원 정보를 자동 추출하고, 이를 서버로 전달해 정보를 관리·활용하는 시스템입니다. 평소 즐기던 게임의 파티 모집 과정을 자동화하고 싶다는 생각에서 출발해 개인 프로젝트로 시작했습니다.

초기에는 JavaScript 기반 Tesseract.js와 OpenCV.js를 활용하여 클라이언트 OCR 처리를 직접 구현했으나, 한글 및 게임 특수폰트 인식에서 정확도에 한계를 느꼈습니다. 전문적인 OCR 학습과 전처리 기술이 필요하다고 판단해, 관련 기술을 가진 팀원을 직접 모집하고 함께 협업 구조를 설계했습니다.

저는 프로젝트의 기획 및 전체 시스템 구조 설계, 프론트-백 연동 , API 처리 등을 담당했으며, OCR 정확도 향상을 위한 실험과 UI 개선을 주도했습니다. 팀원들과는 역할을 분담하고 주기적인 회의를 통해 목표를 공유하며 개발을 이어갔습니다.

이 프로젝트를 통해 혼자서 끝까지 밀어붙이는 추진력과, 한계를 인정하고 협업을 선택할 줄 아는 유연함, 그리고 협업에서의 커뮤니케이션 감각을 키울 수 있었습니다

## 다른 사람과 구별되는 나만의 강점
저는 문제가 생겼을 때 다양한 관점에서 접근하며, 여러 해결 방안을 스스로 설계하고 그중 최선의 선택지를 찾아 실행하는 능력을 갖추고 있습니다.

프로젝트 ‘로스트파티’를 진행하며 여러 기술적·구조적 문제에 직면했을 때, 단순한 해결보다는 여러 대안을 나열하고 각각의 장단점을 분석한 뒤, 상황에 가장 적합한 방법을 팀원들과 함께 선택해 실행해왔습니다. 예를 들어, 클라이언트 처리 vs 서버 처리, 직접 구현 vs 외부 API 연동 등 여러 갈래의 선택지 속에서 문제해결 방식의 장단점을 균형 있게 고려해 판단했습니다.

이처럼 저는 문제를 복잡하게 만들지 않으면서도 다양하게 바라보고, 해결책을 체계적으로 정리한 후 실행에 옮기는 데 강점이 있습니다. 조직 내에서 ‘기획-판단-실행’을 연결하는 사람으로서 기여할 수 있다고 자신합니다.

## 성장 방향

**"기술과 비즈니스를 연결하는 개발자로 성장"**

현재 풀스택 개발 역량과 클라우드 인프라 경험을 바탕으로, 확장 가능한 시스템 아키텍처를 설계하고 구현하는 개발자로 성장하고 있습니다. 

**핵심 역량 강화 방향**
- **클라우드 아키텍처 전문성**: AWS ECS, RDS, S3 등 고급 클라우드 서비스로 확장하여 대규모 시스템 운영 경험 축적
- **시스템 안정성 및 보안**: 모니터링, 백업, 보안 체계 구축을 통한 프로덕션 환경 운영 전문성 확보
- **팀 기술 리더십**: 협업 중심의 개발 문화 구축과 기술적 의사결정을 주도하는 개발자로 성장
